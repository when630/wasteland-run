## Step 1: 시드 RNG (Feature 7)

모든 랜덤 결과의 기반. 먼저 구현해야 이후 콘텐츠가 재현 가능.

### 1.1 `frontend/src/utils/rng.ts` — Mulberry32 PRNG 추가

```typescript
export class SeededRNG {
  private state: number;
  constructor(seed: number) { this.state = seed; }
  next(): number { /* mulberry32 */ }
  nextInt(max: number): number;
  shuffle<T>(arr: T[]): T[];
}
export function hashString(s: string): number; // 문자열 → 숫자 시드 변환
```

- `customShuffle(arr, rng?)` — 선택적 RNG 파라미터 추가 (기존 Math.random 폴백)
- `generateUniqueId()` — 고유성 보장 위해 카운터 접미사 사용 (시드 RNG 아닌 기존 방식 유지)

### 1.2 `frontend/src/store/useRngStore.ts` — RNG 싱글톤

`useRunStore.runSeed`에서 초기화. 서브시스템별 분리된 RNG 인스턴스:
- `intentRng`, `eventRng`, `shuffleRng`, `lootRng`, `mapRng`

### 1.3 `Math.random()` 교체 대상 (10개 파일)

| 파일 | 교체 내용 |
|------|-----------|
| `enemies.ts` | `determineNextIntent(baseId, rng)` 시그니처 변경 |
| `events.ts` | 모든 `onSelect` 내 `Math.random()` → `eventRng.next()` |
| `useMapStore.ts` | `generateMap()` 내부 → `mapRng` |
| `BattleView.tsx` | 적 선택 랜덤 → `battleRng` |

## Step 4: 전투 비주얼 폴리시 (Feature 3)

### 4.1 화면 흔들림 (Screen Shake)

`BattleStage.tsx` 루트 Container에 `shakeOffsetX/Y` 적용. 플레이어 피격 시 300ms 감쇄 진동.
- `useBattleStore`에 `screenShake: number` 추가 (또는 BattleStage 로컬 상태)
- `playerHitQueue` 소비 시 트리거

### 4.2 떠다니는 데미지 숫자

**신규: `components/pixi/DamageNumber.tsx`**
- Props: `amount, x, y, color, onComplete`
- 애니메이션: 800ms 동안 60px 상승 + alpha 1→0 페이드
- 색상: 빨강(물리), 주황(화상), 초록(독/힐), 보라(특수)

`useBattleStore`에 `damageNumberQueue: Array<{id, amount, x, y, type}>` 추가.
적 데미지/플레이어 데미지 시 큐에 push, BattleStage에서 렌더 후 제거.

### 4.3 멀티히트 캐스케이드

다단 히트 시 150ms 간격으로 데미지 넘버 스태거링. `executeAction` 내 멀티히트 반복마다 딜레이된 큐 엔트리.

### 4.4 상태이상 아이콘 뱃지

`AnimatedEnemy.tsx`에서 적 스프라이트 위/아래에 컬러 뱃지:
- BURN: 주황 원 + 스택수, POISON: 초록 원 + 스택수
- VULNERABLE: 분홍 + 스택수, WEAK: 파랑 + 스택수
- Pixi Text로 렌더, 기존 텍스트 대체

### 4.5 카드 호버 예상 데미지

`damageCalculation.ts`에 `calculatePreviewDamage(card, enemies, state)` 추가.
`Hand.tsx`에서 호버 시 "~X 데미지" 오버레이 표시.

**파일 변경: 1 신규 (DamageNumber.tsx), 5 수정 (BattleStage, AnimatedEnemy, Hand, damageCalculation, enemySlice 또는 playerSlice)**
