# 📝 게임 기획서 (GDD): 웨이스트랜드 런 (Wasteland Run - 가제)

## 1. 게임 개요 (Game Overview)
* **장르:** 덱 빌딩 로그라이크 RPG
* **플랫폼:** Web Browser (PC/Mobile 크로스 플레이 지향)
* **시각 요소:** 2D 사이드뷰 (Side-view), 거칠고 절망적인 분위기의 레트로 픽셀 아트
* **핵심 플레이:**
  * 왼쪽의 플레이어 캐릭터와 오른쪽의 적들이 대치하는 구도.
  * 적의 '의도'를 파악하고, 이원화된 자원(AP/탄약)과 5가지 타입의 카드를 전략적으로 사용하여 생존하는 퍼즐 같은 전투.

---

## 2. 핵심 메커니즘 (Core Mechanics)

### 2.1. 사이드뷰 전투 시스템
* 플레이어는 왼쪽 고정, 적들은 오른쪽에 배치 (최대 3~4마리 멀티 배틀 지원).
* 모든 공격과 방어, 카드 사용 이펙트는 좌우 상호작용으로 직관적으로 연출.

### 2.2. 이원화된 자원 시스템
* **행동력 (Action Point - AP):** 매 턴 3칸씩 주어지며 턴 종료 시 소멸. (물리 공격/방어 카드 사용에 필수)
* **탄약 (Ammo):** 전투 중 누적되며 턴이 지나도 유지됨. (강력한 '특수 공격' 카드 사용에 소모)

### 2.3. 5원소 카드 시스템
이 게임의 덱은 다음 5가지 분류의 카드로 구성되며, 각 카드 간의 명확한 상성과 연계가 핵심.

| 타입 | 명칭 | 주요 자원 | 특징 및 예시 (사이드뷰 연출) |
| :-- | :--- | :---: | :--- |
| **1** | **물리 공격** (Slash/Blunt) | **AP** | **특성:** 적의 물리 방어도를 깎거나 **'탄약'** 수급.<br>**연출:** 플레이어가 적에게 돌진하여 타격하거나 원거리 투척. (예: `쇠파이프 후려치기`, `단검 투척`) |
| **2** | **특수 공격** (Fire/Chem/Electric) | **탄약** | **특성:** 탄약을 소모하여 높은 고정 피해. 물리 방어도 무시.<br>**연출:** 화기 발사, 폭발, 화학 물질 살포. (예: `급조된 화염병`, `샷건 사격`, `방사능 빔`) |
| **3** | **물리 방어** (Armor) | **AP** | **특성:** 턴 종료 시 소멸하는 '일반 방어도' 생성. 물리 피해를 막음.<br>**연출:** 방패를 들어 올리거나 고철 장갑을 덧댐. (예: `고철 덧대기`, `엄폐`) |
| **4** | **특수 방어** (Resist) | **AP** | **특성:** 턴 종료 시 소멸하는 '특수 방어도' 생성. 특수(화염/독/방사능) 피해를 막음.<br>**연출:** 방독면 착용, EMP 필드 전개, 방호복 착용. (예: `낡은 방독면`, `전자기장 방출`) |
| **5** | **변화** (Utility/Status) | **혼합** | **특성:** 버프/디버프. 적에게 상태이상을 걸거나, AP/탄약을 펌핑.<br>**연출:** 약물 주입, 연막탄 전개, 고철 루팅. (예: `전투 자극제 투여`, `연막탄`, `강화 숫돌`) |

### 2.4. 적의 의도 (Intent System)
* 적은 자신의 머리 위에 다음 턴 행동을 예고함.
* **물리 의도(⚔️):** 물리 방어로 막아야 함.
* **특수 의도(☣️):** 특수 방어로 막아야 함.
* 플레이어는 코스트 내에서 물리 방어와 특수 방어를 적절히 섞어서 써야 생존 가능.

## 3. 게임 플레이 루프 (Gameplay Loop)

1. **맵 노드 선택:** 슬레이 더 스파이어 스타일의 분기형 지도에서 다음 이동할 구역(전투, 엘리트, 상점, 이벤트, 휴식처) 선택.
2. **전투 페이즈 (사이드뷰):** 매 턴 5장의 카드를 뽑고 코스트 내에서 카드 사용 후 턴 종료. 적 전멸 시 승리.
3. **보상 및 덱 빌딩:** 전투 승리 시 새로운 카드 1장(3장 중 택 1)과 골드 획득. 확률적으로 유물 획득.
4. **성장과 정비:** 상점에서 카드 구매/삭제, 휴식처에서 카드 강화/체력 회복을 통해 덱 정비.
5. **보스전:** 각 맵의 끝에서 지역 보스를 물리치면 다음 챕터로 진행.

---

## 4. 기술 스택 및 개발 환경 (Web Focus)

* **클라이언트 (Web):**
  * **프레임워크/엔진:** React + Pixi.js 또는 Phaser 3
  * **언어:** TypeScript
* **백엔드 (API Server):**
  * **프레임워크:** Spring Boot (Java)
  * **역할:** 플레이어 로그인, 게임 데이터 인증, 런(Run) 기록 저장, 리더보드 관리
* **데이터베이스:**
  * **DB:** MySQL (랭킹, 유저 데이터 저장) + Redis (실시간 캐싱)

## 5. MVP (최소 기능 제품) 개발 마일스톤

1. **1단계: 핵심 전투 프로토타입**
   * FE: 사이드뷰 전투 화면 구성, 기본 AP 시스템, 물리 공격/방어 카드 구현.
   * BE: 로그인 기능, 전투 결과 저장 API.
2. **2단계: 5원소 시스템 및 탄약 메커니즘**
   * FE: 탄약 시스템(물리 공격 획득, 특수 공격 소모) 및 특수 공격/방어, 상태이상 카드 구현.
3. **3단계: 적 AI 및 의도 구현**
   * FE: 적의 '의도' 아이콘 표시 시스템 및 기본 공격 패턴 구현.
   * BE: 적 몬스터 데이터 베이스 구축.
4. **4단계: 로그라이크 루프 (맵 & 덱 빌딩)**
   * FE: 분기형 맵 시스템, 전투 보상 화면, 상점 및 휴식처 기능.
   * BE: 랜덤 맵 생성 알고리즘, 유저 덱 상태 관리 API.
5. **5단계: 콘텐츠 확장 및 런칭 준비**
   * FE/BE: 엘리트/보스 몬스터, 유물 시스템, 무작위 이벤트, 리더보드 추가.
