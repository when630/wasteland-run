# 🚀 Phase 1 - Week 1: 뷰 컴포넌트 뼈대 구조 및 로직 완성

## 1. 개요
* **날짜:** 2026-03-03
* **작업 내용:** React 기반 상태(Zustand)와 UI 레이아웃 구축, Pixi.js 기반 렌더링 캔버스 연동

## 2. 작업 상세 내용
* **Zustand 핵심 스토어 세팅:**
  * `useBattleStore.ts`: 플레이어 AP, 탄약, 턴 카운터 및 인게임 로직 생성
  * `useRunStore.ts`: 플레이어 HP, 재화 정보 등 게임 전반부 상태 정의
* **React UI 구획 분할:**
  * `HUD.tsx`: 상단 상태(체력, 방어도, 특수 방어도, 맵) 바 렌더링
  * `ResourcePanel.tsx`: 좌측 하단 AP/Ammo 및 턴 시스템 버튼 통제
  * `Hand.tsx`: 소지 카드(정렬 영역) 인터렉션 껍데기 구축
* **하이브리드 렌더링 (React + Pixi.js) 세팅:**
  * **`BattleStage.tsx`:** 캔버스 내부 전투 영역. (임시 적군, 아군 텍스처, 텍스트) 상태 변경에 따른 렌더링 반응형 대응 (Pixi.js v7 버전 호환성 기반 TypeScript 에러 수정 적용)
  * **`BattleView.tsx`:** `pointerEvents`, `zIndex` 트릭을 활용해 Canvas 배경 위에 React Component를 얹는 16:9 메인 화면 구성
* **전역 설정:**
  * `React-Router-Dom` App.tsx 연결
  * `index.css` 초기화 패딩 리셋 적용

## 3. 남은 과제 / 이슈
* `@pixi/react` 컴포넌트 내부에서 v8 최신 Pixi의 `Texture`, `TextStyle` 타입 임포트 시 충돌하는 고질적 이슈가 있어 `v7.3.3` 다운그레이드 및 `useMemo` & 타입 단언(`as any`)으로 급한 불을 끄고 빌드 안정성을 확보했습니다. 차후 직접 `useTick` 등을 사용해 PIXI 앱을 바닐라로 통제하는 방안도 고려할 만 합니다. 
* 다음 단계에서는 이 껍데기에 실제 카드의 데이터 구조를 물리는 작업을 진행할 예정입니다.
