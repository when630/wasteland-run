# 🚀 Phase 2 - 몬스터 다양화 및 타겟팅(1:N) 시스템 구축 완료

## 1. 개요
* **날짜:** 2026-03-03
* **작업 내용:** 다수의 적(1:N) 등장 및 플레이어의 타겟팅 2-Step 조작계(카드 선택 -> 적 지정) 구현.

## 2. 작업 상세 내용
* **신규 몬스터 추가 (`assets/data/enemies.ts`):** 
  * `acid_dog` (산성 침 들개): 체력 25. 공격력이 약하지만(4~6), 향후 디버프 중첩 로직을 끼워넣기 좋은 패턴의 저체력 유닛.
  * `waste_slime` (폐기물 슬라임): 체력 60. 확률적으로 공격(3) 대신 자신에게 방어도(5)를 올리는 `BUFF` 인텐트 보유. (추후 배치 예정)
* **2-Step 타겟팅 모드 (`useBattleStore`, `useCardPlay.ts`):**
  * 공격 카드(`DAMAGE`, `DEBUFF` 효과 보유)를 클릭하면 수치가 즉시 차감되지 않고, 스토어 상태가 `targetingCardId` 모드로 전환됨.
  * 화면 상에 **"공격 대상을 클릭하세요!"** 메시지 출력.
  * 대상 지정이 필요 없는 카드(쉴드 획득, 탄약 보충 등)는 타겟팅 모드에 돌입하지 않고 바로 즉시시전됨.
* **배틀 스테이지 다중 렌더링 (`BattleStage.tsx`):**
  * `enemies.map` 스크립트를 통해 화면 우측 상하에 여러 몬스터를 자동 배치(`y` 좌표 오프셋 활용).
  * 타겟팅 모드일 경우 마우스 커서를 `crosshair`로 변형. 각 몬스터 객체(`Container`)에 onClick 이벤트를 심어 클릭 시 지정한 몬스터 ID 매개변수와 함께 데미지 부여 프로세스 진입.
  * 체력이 0 이하가 된 몬스터는 즉시 화면 렌더링 배열에서 숨김 처리됨.

## 3. 남은 단상
* "누구를 먼저 죽여야 내 피해가 최소화될까?"라는 전략적 기틀이 완성되었습니다.
* 체력이 0 이하인 적에 대한 승리/턴 종료 로직(모든 적이 죽었을 때 스테이지 클리어)과, 투박한 디자인의 몬스터들을 예쁜 픽셀 아트로 교체하는 시각화 작업만 남았습니다.
