## 2. Phase 1 — 순수 로직 모듈 추출

거대 스토어/훅에서 순수 함수를 별도 `logic/` 모듈로 분리. 기존 시그니처 변경 없음.

### 2.1 `logic/cardUpgrades.ts`
* `useDeckStore.ts`의 152줄 switch문 → `UPGRADE_TABLE` 룩업 맵 + `applyUpgrade(card)` 함수로 전환.
* `useDeckStore.ts`: 304줄 → ~160줄.

### 2.2 `logic/relicEffects.ts`
* 7개 파일에 산재한 `relics.includes(...)` 20곳+를 트리거 포인트별 순수 함수로 통합.
* 주요 함수: `onBattleStart()`, `onBattleReset()`, `onRestOrEventEnter()`, `onEnemyKilledBySpecial()`, `onFatalDamage()`, `onCardPlayed()`.
* 수정 대상: `BattleView.tsx`, `useBattleStore.ts`, `RestView.tsx`, `EventView.tsx`, `useCardPlay.ts`.

### 2.3 `logic/damageCalculation.ts`
* `useBattleStore.ts`의 `applyDamageToEnemy()` 내부 데미지 수학을 추출.
* `calculateDamageToEnemy()` — VULNERABLE, 방패/레지스트 흡수 처리.
* `calculateDamageToPlayer()` — 다중 히트, 방어, 히트 큐 생성.

### 2.4 `logic/enemyTurnLogic.ts`
* `executeOneEnemyTurn()` (203줄)에서 추출.
* `processStatusDamage()` — 화상/독 틱 데미지.
* `parseAttackIntent()` — 다중 히트 파싱 + 디버프 보정.
* `processStatusDecay()` — 턴 종료 상태이상 감소.

### 2.5 `logic/cardEffectHandlers.ts`
* `useCardPlay.ts`의 157줄 이펙트 루프 → `resolveCardEffects()` + `EffectAction` 판별 유니온(13종) 패턴으로 전환.
* `useCardPlay.ts`: 351줄 → ~270줄.

## 6. Phase 5 — 백엔드 개선

### 6.1 `RunData.updateFrom(RunSaveRequestDto)` — DTO 기반 업데이트
* 16개 위치 파라미터 → DTO 직접 수용 메서드로 교체. 순서 실수 위험 제거.

### 6.2 User 조회 중복 제거
* `UserService.findByUsernameOrThrow()` 추가.
* `RunService`, `LeaderboardService`, `UserStatsService`에서 `UserRepository` 직접 의존 제거.

### 6.3 커스텀 도메인 예외
* `BusinessException` (base) → `UserNotFoundException` (404), `DuplicateUsernameException` (409), `RunDataNotFoundException` (404).
* `GlobalExceptionHandler`에 `BusinessException` 핸들러 추가.

### 6.4 `RunResponseDto`에 `@Builder` 적용
* 위치 기반 생성자 → 빌더 패턴의 `of()` 팩토리 메서드.

### 6.5 설정 외부화
* CORS origin → `application.yml`의 `app.cors.allowed-origins` (쉼표 구분 다중 오리진 지원).
* JWT 만료시간 → `jwt.expire-time-ms` 프로퍼티.
